# CD-12 — Fix argocd-sync.yml: bump per-image tags

[Shared preamble — see README.md]

## Goal
The current `.github/workflows/argocd-sync.yml` opens a PR bumping a single global `image.tag` field in `infra/argocd/overlays/dev/values.yaml`. That works for charts that share the tag (api/workers/frontend) but **silently leaves tile-server stuck on its old tag**. This PR fixes the bump logic to update per-image tags.

## Files to change
- `.github/workflows/argocd-sync.yml` — replace the Python regex block with a structured YAML edit.
- `infra/argocd/overlays/dev/values.yaml` — restructure into a per-image map: `images: { api: { tag: ... }, workers: { tag: ... }, frontend: { tag: ... }, tileServer: { tag: ... } }`.
- `infra/argocd/overlays/staging/values.yaml`, `infra/argocd/overlays/production/values.yaml` — same restructure.
- `infra/helm/api/values.yaml`, `infra/helm/workers/values.yaml`, `infra/helm/frontend/values.yaml`, `infra/helm/tile-server/values.yaml` — update `image.tag` defaults to read from the overlay's `images.<chart>.tag`.
- `infra/argocd/appsets/services.yaml` — pass overlay values through correctly (`helm.valueFiles` may need a tweak).

## Tasks
1. Restructure overlay values into a per-image map. Existing `image.tag` references become per-chart entries.
2. Workflow: use `yq` (Mike Farah's, v4) instead of the Python regex — atomic, validates YAML, and won't accidentally bump unrelated fields:
   ```yaml
   - name: Bump dev image tags
     run: |
       for img in api workers frontend tileServer; do
         yq -i ".images.${img}.tag = \"${{ steps.sha.outputs.short }}\"" "$OVERLAY"
       done
   ```
3. Verify each chart's `image.tag` template reference (`{{ .Values.image.tag | default .Values.global.images.<chart>.tag }}` or similar) resolves correctly.
4. Update `services.yaml` AppSet so the overlay values file is loaded with global scope (not chart-local).
5. PR-create step body lists all four images with their tags individually.

## Out of scope
- Don't add ArgoCD Image Updater (still a P2).
- Don't change which env auto-bumps (still dev only).
- Don't add a CI gate that smoke-tests the rendered chart — leave to a follow-up.

## Definition of done
- `helm template` for all four charts uses the new per-image tag path.
- A test push to main results in a single PR with four tag bumps, one per image.
- The bumped PR merges cleanly and ArgoCD syncs each chart to the new image.
