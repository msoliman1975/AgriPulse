# CD-15 ‚Äî Cost guardrails: budget alarm, EBS sweeper, idle-pod alerts

[Shared preamble ‚Äî see README.md]

## Goal
Three small things that pay for themselves the first time they fire:
1. AWS Budgets alarm at $300/mo with SNS ‚Üí Slack so you know if costs are climbing.
2. A Lambda that snapshots-then-deletes orphan EBS volumes older than 7 days (CNPG and Karpenter both leave detached volumes around in failure modes).
3. Prometheus rules that flag pods with `<2 %` CPU + `<10 %` memory utilisation over 24 hours.

## Files to change
- `infra/terraform/budgets.tf` ‚Äî new. `aws_budgets_budget` + `aws_sns_topic` + Slack subscription via chatbot or webhook lambda.
- `infra/terraform/ebs-sweeper.tf` ‚Äî new. Lambda + EventBridge daily schedule + IAM role.
- `infra/terraform/ebs-sweeper-lambda/` ‚Äî new directory with the Python source (~50 lines).
- `infra/helm/shared/templates/prometheusrule-cost.yaml` ‚Äî new. `PrometheusRule` with two alerts.
- `infra/argocd/overlays/{dev,staging,production}/values.yaml` ‚Äî enable the rule (defaults to disabled in dev to avoid noise from the smaller cluster).

## Tasks
1. AWS Budget:
   - Monthly budget at $300, type `COST`, action when 80 % forecasted + 100 % actual.
   - SNS topic with a Slack webhook subscription (the Slack webhook URL stored in SM, looked up by the Lambda that publishes to Slack ‚Äî or use AWS Chatbot for first-party Slack delivery).
2. EBS sweeper Lambda:
   - Runtime python3.12.
   - List all `available` (unattached) EBS volumes.
   - For each, check tag `created` or `CreatedTime`; if > 7 days old AND not tagged `keep=true`:
     - Create a snapshot tagged `purpose=pre-sweep`.
     - Delete the volume.
   - Dry-run mode behind an env var so first deploy is observation-only.
   - CloudWatch logs.
   - EventBridge schedule: daily at 03:00 UTC.
3. Prometheus rules:
   - `IdlePodCPU`: `rate(container_cpu_usage_seconds_total{namespace!~"kube-.*|argocd"}[24h]) < 0.02` for 24h ‚Üí warning.
   - `IdlePodMemory`: `container_memory_working_set_bytes / container_spec_memory_limit_bytes < 0.10` for 24h ‚Üí warning.
   - Annotations include the cost-impact hint and `runbook_url`.

## Out of scope
- Don't add AWS Cost Anomaly Detection ‚Äî overlapping with Budgets for V1.
- Don't add per-namespace cost allocation. Tagging is enough.
- Don't change EKS instance types to "save money" reactively. Karpenter's consolidation handles that.

## Definition of done
- `terraform apply` creates the budget, SNS topic, Lambda, EventBridge rule.
- Lambda runs in dry-run mode the first day and logs would-be deletions to CloudWatch ‚Äî verify with `aws logs tail /aws/lambda/ebs-sweeper --follow`.
- Prometheus rules appear in the kube-prometheus operator's loaded rules.
- A deliberate $0.01 test charge above the threshold (or temporarily lower the threshold) fires the Slack alert.
- PR description includes the verification commands and instructions for flipping the sweeper out of dry-run mode after 1 week of observation.
