# CD-10 — ArgoCD bootstrap via Terraform + AppSets sync

[Shared preamble — see README.md]

## Goal
Install ArgoCD itself via Terraform (one-time bootstrap), apply the four AppSets (`bootstrap`, `platform`, `services`, `observability`), and expose the ArgoCD UI at `argocd.agripulse.cloud`. After this PR, ArgoCD is the boundary: everything in `infra/argocd/` and `infra/helm/` flows through GitOps, no more manual `kubectl apply`.

## Files to change
- `infra/terraform/argocd.tf` — new. `helm_release "argocd"` with custom values.
- `infra/terraform/locals.tf` — add `argocd_hostname = "argocd.agripulse.cloud"`.
- `infra/argocd/values/argocd-server.yaml` — new. Server config: ingress enabled, hostname, RBAC defaults, OIDC config (stub for now — Keycloak SSO wires in CD-13).
- `infra/argocd/appsets/bootstrap.yaml` — verify it targets the right repo + paths.
- `docs/runbooks/argocd-bootstrap.md` — new, ~50 lines.

## Tasks
1. Terraform `helm_release`:
   - Chart `argo/argo-cd` pinned to a specific version (~7.x).
   - Custom values reference `infra/argocd/values/argocd-server.yaml`.
   - Depends on the EKS module + the External Secrets bootstrap (so admin password can be read from SM).
2. ArgoCD server values:
   - `server.ingress.enabled: true`, hostname `argocd.agripulse.cloud`, ingressClassName `nginx`, cert-manager annotation for letsencrypt-prod.
   - `server.ingress.annotations.nginx.ingress.kubernetes.io/whitelist-source-range: <maintainer-ip>/32` initially (lock down until Keycloak SSO lands).
   - `configs.params.server.insecure: false` (TLS terminates at the ingress, ArgoCD speaks gRPC-web behind it).
   - `controller.replicas: 2`, `repoServer.replicas: 2`.
   - Admin password sourced from SM via External Secret (chicken-and-egg solved by Terraform seeding the SM entry on first apply with a generated value, then ExternalSecret syncs it).
3. After `terraform apply`, manually `kubectl apply -f infra/argocd/appsets/bootstrap.yaml` ONCE (or have Terraform `kubectl_manifest` apply it). Bootstrap AppSet then sources `platform`, `services`, `observability` AppSets, which source individual Applications. From here, ArgoCD is self-managing.
4. Runbook covers:
   - Initial login: get admin password from SM, log in via UI.
   - Sanity check: all Applications in Synced + Healthy state.
   - How to add a new chart: drop it in `infra/helm/<name>/`, add entry to `services` AppSet, push.
   - How to add a new env: extend `services` AppSet's `generators.list` with a new env, add overlay.
   - Failure mode: "Application stuck in OutOfSync after a PR" — how to diff, retry, hard-refresh.
   - How to recover ArgoCD itself if its Deployment is broken (`kubectl rollout undo` on `argocd-server`).

## Out of scope
- Don't wire Keycloak SSO yet (CD-13 does that).
- Don't add Image Updater. The PR-bump flow from `argocd-sync.yml` is the contract.
- Don't enable `applicationSet.replicas > 1` — the controller doesn't support active-active.

## Definition of done
- `terraform apply` installs ArgoCD; `kubectl get pods -n argocd` shows all Running.
- `https://argocd.agripulse.cloud` resolves, returns a real cert, login works.
- Bootstrap AppSet visible in the UI; child AppSets visible; all underlying Applications Synced.
- Runbook reviewed.
