## Goal
Finalise `infra/argocd/overlays/production/values.yaml` with production-shaped numbers (replicas, resources, retention) and write the manual promotion + smoke-test runbook for taking dev ‚Üí staging ‚Üí prod.

## Files to change
- `infra/argocd/overlays/production/values.yaml` ‚Äî finalise.
- `infra/argocd/overlays/staging/values.yaml` ‚Äî finalise.
- `docs/runbooks/first-prod-deploy.md` ‚Äî new, ~100 lines.
- `docs/runbooks/promotion-procedure.md` ‚Äî new, ~40 lines.

## Tasks
1. Production overlay:
   - `replicaCount` for api/frontend: 3 (HA across 3 AZs).
   - `replicaCount` for workers: light=2, heavy=1 (heavy can scale on demand via HPA).
   - `postgresCluster.instances: 3` (HA Postgres across AZs).
   - `postgresCluster.storage.size: 100Gi`, `walStorage.size: 20Gi`.
   - `postgresCluster.backup.enabled: true`, `retentionPolicy: 30d`.
   - Resources: bigger CPU/memory requests than dev. api: 500m/512Mi req, 2/2Gi limit. Workers: 500m/1Gi req, 2/4Gi limit (heavy gets more).
   - `ingress.annotations.cert-manager.io/cluster-issuer: letsencrypt-prod`.
   - `ingress.host` patterns: `<service>.agripulse.cloud` (no env prefix).
   - HPA: api enabled, min=3 max=10, target CPU 70%.
   - PDB: api minAvailable=2, workers minAvailable=1.
2. Staging overlay:
   - Same shape as prod but `replicaCount=2`, smaller resources, `letsencrypt-staging`, hostnames `<service>.staging.agripulse.cloud`, `postgresCluster.instances: 2`.
3. First-prod-deploy runbook covers (in order):
   - Pre-flight: confirm dev + staging are Healthy + Synced for 24h, no Sentry/GlitchTip alerts.
   - Manual seed of production secrets in SM (CD-9 runbook reference).
   - First sync: open the prod tag-bump PR, merge, watch ArgoCD.
   - Migrations: confirm the PreSync Job ran.
   - Smoke: `curl https://api.agripulse.cloud/health` ‚Üí 200, log in via Keycloak, create a tenant, run one of each module's primary flow.
   - DNS propagation timing (TTL 300s on prod records ‚Äî wait 5 min before claiming green).
   - Rollback: revert the tag-bump PR, ArgoCD syncs the previous SHA, migrations are forward-only ‚Äî document the manual `alembic downgrade` if needed.
4. Promotion procedure runbook:
   - Dev ‚Üí staging: human-authored PR bumping `staging/values.yaml` `images.*.tag`. CODEOWNERS requires one approval.
   - Staging soak: 24‚Äì72h depending on change risk.
   - Staging ‚Üí prod: same PR-bump pattern. 2 approvals.
   - Hotfix path: skip staging for sev-1 only, document the gate.

## Out of scope
- Don't enable canary or blue/green deploys. ArgoCD rolling is V1.
- Don't add automated promotion. Manual is the feature.
- Don't add Slack notifications for ArgoCD sync events here ‚Äî that's an observability follow-up.

## Definition of done
- `helm template` for every chart against the prod overlay renders without errors.
- Both runbooks reviewed.
- The first-prod-deploy runbook has been walked through verbally with one person who didn't write it.
