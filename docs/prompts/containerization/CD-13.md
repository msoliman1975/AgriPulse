## Goal

## Tasks
1. Keycloak chart values:
   - `replicas: 2` (uses Infinispan distributed cache).
   - `cache: ispn` (default in modern Keycloak).
   - `db.vendor: postgres`, `db.url`, `db.username`, `db.password` from External Secret pointing at SM (`agripulse/<env>/keycloak-db-password` ŌĆö add to CD-9 if not there).
   - `hostname.hostname: auth.agripulse.cloud` (prod), `auth.dev.agripulse.cloud` (dev).
   - `hostname.strict: true` (issuer URL stable).
   - `ingress.enabled: true`, cert-manager annotation, ingressClassName.
   - `proxy: edge` (TLS terminates at ingress-nginx).
2. CNPG: create a `keycloak` database via either a `postInitSQL` entry (simpler) or a CNPG `Database` CR (cleaner; cnpg.io/v1 `Database`). Owner user `keycloak` with a password from SM.
3. Realm JSON: export the working dev realm with `kc.sh export --realm agripulse --file /tmp/realm.json`, sanitize secrets out (clients with `secret` fields ŌåÆ reference SM), commit to `infra/helm/keycloak/realms/agripulse-realm.json`.
4. The realm ConfigMap mounts the JSON; Keycloak `--import-realm` imports on first start.
5. Runbook covers:
   - How to edit the realm: edit JSON in repo, PR, ArgoCD syncs, **must** force a rolling restart or `--import-realm` does nothing on running pods (alternative: use the admin API).
   - How to add a client / role.
   - How to rotate the admin password (SM put ŌåÆ restart).
   - How to recover from a corrupted realm (delete the CNPG `keycloak` database, restart Keycloak with `--import-realm`).
