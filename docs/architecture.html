1 Overview & pillars

AgriPulse is a cloud-native SaaS that lets commercial agribusinesses in Egypt monitor crop health, irrigation, and weather using satellite imagery, weather forecasts, and custom signals; with rules-based alerting and explainable agronomic recommendations. The platform is a modular monolith backend plus a single-page web frontend and a dedicated tile server, deployed on Kubernetes.

Multi-tenant

Bridge model: shared infra, schema-per-tenant in a single Postgres. Tenant resolved from JWT claim, never URL.

Cloud-portable

Kubernetes (EKS), CloudNativePG, MinIO/S3 abstraction, no AWS-locked managed services in the core.

Capability RBAC

Eight roles, three-layer scope resolution (platform â†’ tenant â†’ farm). Code uses @requires_capability.

i18n + RTL

English + Arabic at launch, logical CSS, react-i18next, language & unit preferences per user.

Egypt-native

Feddan as primary unit, governorate geography, salinity-aware soil, Nile/canal/well water sources.

Subscription

Manual invoicing in MVP; Stripe/Paymob in P2.

5 Tenancy model

Schema layout

public shared catalogs, tenants, users, RBAC, pgstac.

tenant_<uuid> all tenant-owned data: farms, blocks, indices aggregates, alerts.

Cross-schema FKs are logical, not declarative â€” the app enforces them; a periodic consistency-check job logs orphans.

Resolution

Tenant resolved only from the JWT tenant_id claim. URL paths and query parameters are never authoritative.

FastAPI middleware sets search_path per transaction. RLS policies on shared tables that contain tenant FKs back this up.

Migrations

Two Alembic trees: migrations/public/ for the shared schema and migrations/tenant/ applied per-tenant by a custom runner so new tenants get a clean schema and existing tenants are upgraded in lock-step.

9 Imagery pipeline

Provider

Sentinel Hub Process API behind SentinelHubProvider. Adapter pattern leaves room for a future Sentinel2OpenDataProvider and PlanetScopeProvider.

Indices

Hybrid: six standard indices (NDVI, NDWI, EVI, SAVI, NDRE, GNDVI) precomputed and stored as COGs + aggregated to TimescaleDB; custom indices computed on-demand.

Storage / projection

COGs in object storage in UTM 36N (EPSG:32636); web tiles served in Web Mercator (EPSG:3857) by TiTiler reprojecting on the fly.

STAC

pgstac inside the same Postgres instance â€” no separate STAC server.

Cloud thresholds

60 % for visualization, 20 % for index aggregation; per-tenant overrides supported.

Retention

90 days S3 Standard, then S3 Glacier Instant Retrieval via lifecycle rule.

Idempotency

Deterministic asset IDs: {provider}/{product}/{scene_id}/{aoi_hash}/{index_or_band}.tif.

Triggers

Celery Beat scheduled polling + on-demand refresh button. No webhooks in MVP.

Pipeline stages

SubscribeAPI records imagery subscriptions for AOIs.
PollBeat polls Sentinel Hub by AOI/cadence.
FetchHeavy worker downloads scene, generates per-index COGs.
CatalogUpserts STAC items in pgstac with deterministic IDs.
AggregateComputes per-block stats â†’ TimescaleDB hypertable.
ServeTiTiler streams tiles; API exposes scenes & timeseries; frontend overlays on MapLibre.

10 Alerts & recommendations

Alerts

  • Tier-2 compound rules (AND/OR groups, time windows, deltas, trends). No arbitrary scripted rules.
  • Pull-based evaluation every 15 min via Celery Beat.
  • Lifecycle: open â†’ acknowledged â†’ resolved (or snoozed â†’ open, or auto_resolved).
  • Channels: in-app (SSE), email (SMTP), webhook (HMAC). No SMS until P2.
  • Cooldown enforced structurally by uniqueness of open alerts per (rule, farm, block).

Recommendations

  • Hand-authored YAML decision trees per crop, versioned and immutable once published.
  • Daily per-block evaluation â€” slower-moving than alerts.
  • Explainability mandatory: every recommendation carries tree_path plus text_en/text_ar.
  • Lifecycle: open â†’ applied | dismissed | deferred | expired.
Single source of truth. The condition evaluator lives in backend/app/shared/conditions/ and is shared by alerts and recommendations. Never reimplement.

Logs

Structured JSON via structlog â†’ Loki. Mandatory fields: timestamp, level, service, correlation_id, tenant_id, user_id (when known). No PII.

15 Out of scope (MVP)

The following must not appear during MVP development without an explicit ADR:
  • Yield/harvest forecasting
  • Mobile offline-first app
  • IoT sensor ingestion (only manual signals in MVP)
  • ExternalAdvisor cross-tenant role
  • Equipment, inventory, financial modules
  • Real billing engine (Stripe / Paymob)
  • Compliance reports (GlobalGAP, FSMA)
  • ML-based field auto-detection (only grid-based auto-blocking)
  • Higher-resolution imagery (Planet, Airbus)
  • SMS notifications
  • Apache Superset
  • Self-managed Sentinel-2 pipeline (only Sentinel Hub API in MVP)
